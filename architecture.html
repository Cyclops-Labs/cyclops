Rating-Charging-Billing
Architecture

The architecture is modular in nature and made up of 3 main micro-services, communicating
with each other using REST APIs and JSON exchange format.

UDR micro-service

UDR micro-service connects to cloud installation such as OpenStack, or CloudStack and autonomously collects the usage data. Harmonised data is being saved into a Time Series Based database and exposed through REST APIs.

Data collection through connectors
Usage data transformation and harmonisation
Availability of usage data records via REST APIs
External SaaS/PaaS usage data insertion via REST APIs

RC micro-service

Dynamic rates are generated through a rule engine by considering the various policies, environmental factors and pricing strategies. A charge for the usage made is calculated by using the harmonised data from UDR micro-service.

Dynamic Rating and Charging
Rule engine based Rating engine
CDR persistence for data redundancy
Exposes CDR through REST APIs

Billing micro-service

Various plans, discounts and coupons are considered through a billing engine to calculate the price to be paid by the user. Once the invoice is generated, the service connects to the payment gateway for payment fulfilment.

Rule engine based Billing engine
Connection to payment gateway
Exposes billed data through REST APIs

Dashboard

Data from the UDR, RC and Billing micro-services is presented to the end user in the form of charts, statistics and bills. Admins can manage user accounts, select what meters are being used and configure the rating details.
